# task-killer

Лёгкий background-агент для контроля запуска процессов в Windows-среде.  
Проект предназначен для использования в управляемой инфраструктуре (учебные/корпоративные станции), где требуется ограничение запуска нежелательного ПО.

---

## 📌 Назначение

Система позволяет централизованно управлять списком запрещённых процессов и автоматически завершать их при запуске на рабочих станциях.

Конфигурация управляется отдельно от бинарного файла и может обновляться без пересборки приложения.

---

## ⚙️ Возможности

- Фоновый агент на Go (lightweight background service)
- Мониторинг активных процессов в реальном времени
- Завершение процессов через WinAPI
- Policy-based конфигурация через JSON
- Обновление конфигурации через SMB-шару
- Запуск через Windows Task Scheduler
- Разделение системных и пользовательских процессов

---

## 🧠 Архитектура

Система построена по простой агентной модели:

**Конфигурация → Агент → Мониторинг процессов → WinAPI → Действие**

### Основные компоненты:
- **Config Loader** — загрузка JSON-конфига с SMB
- **Policy Engine** — обработка правил блокировки
- **Process Monitor** — отслеживание активных процессов
- **WinAPI Layer** — завершение процессов на уровне ОС

---

## 📄 Пример конфигурации

```json
{
    "blacklist":[
        "telegram.exe",
        "steam.exe"
    ],
    "time_cfg_request": "30s",
    "time_idle": "5s",
    "log_path":"../cfg/cfg.json"
}
```

### 🪵 Улучшение логирования
- Переход от `stdout` к асинхронной записи логов в файл
- Реализация отдельного логирующего worker-а (goroutine)
- Буферизация логов для снижения I/O нагрузки
- Поддержка уровней логирования (INFO / WARN / ERROR)
- Возможность ротации лог-файлов (по размеру или дате)

### ⚙️ Расширение policy engine
- Поддержка whitelist + blacklist одновременно
- Группировка правил по пользователям/станциям
- Условные правила (по времени, пользователю, контексту)
